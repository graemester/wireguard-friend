## Design Principles

### Core Values

1. **Clarity Over Cleverness**: Information should be immediately understandable
2. **Progressive Disclosure**: Show essential info first, details on demand
3. **Consistent Interaction**: Same patterns across all interfaces
4. **Fast Feedback**: Immediate visual confirmation of actions
5. **Graceful Degradation**: Works without Rich library, without color, without graphics

### Design Constraints

- **TUI**: Rich library for enhanced rendering, fallback to plain text
- **Color Palette**: Cyan primary, yellow warnings, red errors, dim for metadata
- **Single Keypress**: No Enter required for common actions (1-9, a-z)
- **Alternate Screen**: Clean interface isolation, no terminal history pollution

### Typography Patterns

- **Titles**: UPPERCASE, bold
- **Menu Numbers**: [cyan]N.[/cyan] format
- **Hints**: [dim]- descriptive text[/dim]
- **Status**: [ONLINE], [DEGRADED], [OFFLINE]
- **Keys**: Key: abcd1234... (truncated to 24-30 chars)
- **Breadcrumbs**: [Section] > [Subsection] > Item

### Color Usage

- **Cyan**: Menu numbers, active selections, primary actions
- **Yellow**: Warnings, pending updates, attention needed
- **Red**: Errors, destructive actions
- **Green**: Success messages (âœ“ symbol)
- **Dim**: Metadata, secondary information, back/quit options

## Accessibility Guidelines

### Screen Reader Compatibility (TUI)

**Challenges:**
- Box-drawing characters may be read as individual characters
- ANSI color codes are ignored
- Single-keypress input isn't audible

**Solutions:**

1. **Alternative Text Mode**
   ```bash
   wg-friend --accessible
   # or
   export WGF_ACCESSIBLE=1
   ```

   In accessible mode:
   - No box-drawing characters (use dashes/pipes)
   - No colors (semantic prefixes instead)
   - Explicit prompts for all inputs
   - Verbose descriptions

   Example:
   ```
   MAIN MENU

   Operations:
   1. Dashboard (View system overview)
   2. Manage Peers (View and edit peers)
   3. Network Topology (Visual network map)

   Type a number and press Enter to select.

   Your selection: _
   ```

   ```
    Option 1: Dashboard - View system overview.
    Option 2: Manage Peers - View and edit peers.
    ..."
   ```

### Keyboard Navigation Patterns

**Universal Shortcuts:**
- **Tab**: Move to next field/button
- **Shift+Tab**: Move to previous field/button
- **Enter**: Confirm/Select
- **Escape**: Cancel/Back
- **Space**: Toggle checkbox/Select item
- **Arrow Keys**: Navigate lists
- **Home/End**: Jump to first/last item
- **Ctrl+C**: Interrupt operation (safe)

**List Navigation:**
- **j/k**: Vim-style down/up
- **n/p**: Next/Previous page
- **/** (slash): Search/Filter
- **g**: Go to top
- **G**: Go to bottom

**Consistency Rules:**
1. **Always** provide 'q' or 'b' for quit/back
2. **Always** allow Enter key as alternative to single-keypress
3. **Always** show available keys in footer
4. **Never** use single letters that could be confused (o/0, l/1)

```jsx
// Button Component
const Button = ({
  children,
  variant = 'primary',
  size = 'md',
  onClick,
  disabled = false
}) => {
  const baseClasses = "rounded-lg font-medium transition-colors";
  const variants = {
    primary: "bg-blue-500 hover:bg-blue-600 text-white",
    secondary: "bg-gray-200 hover:bg-gray-300 text-gray-900",
    danger: "bg-red-500 hover:bg-red-600 text-white"
  };
  const sizes = {
    sm: "px-3 py-1.5 text-sm",
    md: "px-4 py-2 text-base",
    lg: "px-6 py-3 text-lg"
  };

  return (
    <button
      className={`${baseClasses} ${variants[variant]} ${sizes[size]}`}
      onClick={onClick}
      disabled={disabled}
    >
      {children}
    </button>
  );
};

// Card Component
const Card = ({ title, children, actions }) => (
  <div className="bg-white dark:bg-slate-800 rounded-lg shadow-md border border-gray-200 dark:border-slate-700">
    {title && (
      <div className="px-6 py-4 border-b border-gray-200 dark:border-slate-700">
        <h3 className="text-lg font-semibold">{title}</h3>
      </div>
    )}
    <div className="px-6 py-4">
      {children}
    </div>
    {actions && (
      <div className="px-6 py-4 border-t border-gray-200 dark:border-slate-700 flex gap-2">
        {actions}
      </div>
    )}
  </div>
);

// Status Indicator Component
const StatusIndicator = ({ status }) => {
  const statusConfig = {
    online: { color: 'bg-green-500', label: 'Online', animate: true },
    offline: { color: 'bg-gray-400', label: 'Offline', animate: false },
    degraded: { color: 'bg-yellow-500', label: 'Degraded', animate: true }
  };

  const config = statusConfig[status] || statusConfig.offline;

  return (
    <div className="flex items-center gap-2">
      <div
        className={`w-3 h-3 rounded-full ${config.color} ${config.animate ? 'animate-pulse' : ''}`}
      />
      <span className="text-sm">{config.label}</span>
    </div>
  );
};

// Usage Example
```

## Conclusion

This comprehensive UI/UX design specification provides a complete blueprint for implementing the features outlined in the WireGuard Friend Innovation Roadmap 2025. The designs maintain consistency with the existing TUI interface while introducing modern web dashboard capabilities.

**Key Design Decisions:**

1. **Dual Interface Strategy**: Maintain powerful TUI for power users, add web dashboard for accessibility
2. **Progressive Disclosure**: Show essential information first, details on demand
3. **Accessibility First**: Screen reader support, keyboard navigation, high contrast modes
4. **Consistent Language**: Same terminology, icons, and interaction patterns across interfaces
5. **Performance Focused**: Fast rendering, efficient updates, responsive interactions
